# NAMING AND STRUCTURE PROTOCOL (⧉✧⚓)
### Deterministic Naming, Titles, Glyphs, and File Structure

This protocol defines the canonical grammar for filenames, titles, glyphs,  
and internal file structure.

---

# 1. Filenames
- UPPERCASE  
- Hyphens only  
- No spaces  
- No underscores  
- Suffixes required:
  - -MODEoptional (Modes use numeric prefix instead)  
  - -CURRENT 
  - -CONNECTOR 
  - -PROTOCOL 

Modes must use:
- NN-NAME.md 

---

# 2. Titles
First line must be:
- # NAME (GLYPH) 

Second line optional:
- ### Subtitle 

---

# 3. Glyph Rules
- Must be unique  
- Must be visually distinct  
- Must be added to RUNESTONES  
- Must be added to Cross‑Layer Map  

---

# 4. Section Ordering
Required order:
1. Title  
2. Subtitle (optional)  
3. Essence  
4. When to Use  
5. Steps (if protocol)  
6. Relations  

---

# 5. Exclusion Rules
The following files must never be included in the artifact:
- README.md  
- OVERVIEW.md  
- INDEX.md  

---

# 6. Deterministic Ordering
- Alphabetical within directories  
- Numeric ordering for Modes  
- Protocols ordered by invocation sequence  

---

# 7. Close
Confirm naming and structure are stable.
